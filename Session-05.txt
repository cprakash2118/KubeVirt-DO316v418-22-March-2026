# Networking Part w.r.t OpenShift vs Other k8s distribution 


Workload w.r.t Container (when deal with OCP)
 -> Deployment -> RS -> POD 
 -> Service (expose respective service) -> route (http/https - default TLS enabled and advance, easy to handle).
 


Workload w.r.t Container (when deal with other k8s)
 -> Deployment -> RS -> POD 
 -> Service (expose respective service) ->  ingress.
 ingress is a standard k8s API object used to expose http/https route to service inside a cluster.	
 -> Requires ingress controller.
    like NGINX/HAproxy ingress controller etc.
 -> TLS supported (manuall effort required)
 -> Portable (Across any k8s distribution)
 
  oc expose service svc_name 
  http://appName-nsName.apps.clusterName.baseDomain   -> 80 
  https://appName-nsName.apps.clusterName.baseDomain  -> 443
